# Contributing Guidelines

Thank you for your interest in contributing to Awesome Agentic Commerce! This guide will help you contribute effectively.

## How to Contribute

### Adding a New Resource

1. **Check for duplicates**: Make sure the resource isn't already listed
2. **Choose the right section**: Place your contribution in the most appropriate section
3. **Follow the format**: Use the existing format for consistency
4. **Provide quality**: Only suggest resources that you have personally used or evaluated
5. **Write clear descriptions**: Keep descriptions concise but informative (1-2 sentences)

### Format Guidelines

#### For Papers

```markdown
- [Paper Title](link-to-paper) - Brief description of the paper's contribution. *Venue Year*
```

Example:
```markdown
- [ReAct: Synergizing Reasoning and Acting in Language Models](https://arxiv.org/abs/2210.03629) - Introduces a paradigm combining reasoning and acting for language models. *ICLR 2023*
```

#### For Frameworks/Tools

```markdown
- [Tool Name](link-to-repo) - Brief description of what the tool does and its main features.
```

Example:
```markdown
- [LangChain](https://github.com/langchain-ai/langchain) - Framework for developing applications powered by language models with agent capabilities.
```

#### For Datasets

```markdown
- [Dataset Name](link-to-dataset) - Description of the dataset, size, and use cases.
```

### Quality Standards

Resources should meet these criteria:

- **Relevance**: Directly related to agentic commerce, AI agents, or commerce AI
- **Quality**: Well-documented, maintained, or peer-reviewed
- **Accessibility**: Publicly accessible (papers, open-source tools, or documented commercial solutions)
- **Actuality**: Recent or still relevant to current practices

### Pull Request Process

1. **Fork the repository** and create a new branch
2. **Make your changes** following the format guidelines
3. **Ensure proper alphabetical ordering** within sections (if applicable)
4. **Write a clear PR description** explaining what you're adding and why it's valuable
5. **Submit the pull request**

### What Not to Include

- Promotional content without genuine value
- Resources that are no longer maintained (unless historically significant)
- Duplicate entries
- Resources behind paywalls (unless exceptional)
- Low-quality or untested resources

## Suggesting Improvements

- **Issues**: Open an issue to suggest new sections or structural changes
- **Discussions**: Start a discussion for broader topics
- **Feedback**: Provide feedback on existing entries if they're outdated or incorrect

## For Survey Paper Contributors

If you're contributing papers for our survey:

1. Ensure papers are relevant to agentic commerce
2. Include proper citations with venue and year
3. Add papers to the most specific applicable section
4. Consider adding a brief note about the paper's relevance to the survey

## Code of Conduct

Please note that this project follows our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.

## Questions?

If you have questions about contributing, feel free to:
- Open an issue with the label `question`
- Reach out to the maintainers

Thank you for making Awesome Agentic Commerce better!
